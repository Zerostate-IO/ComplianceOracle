SECTION 5: TESTING PATTERNS USED
================================================================================

1. Mock-based Unit Testing
   - All tests use mocks to avoid external dependencies
   - No real NIST data or ChromaDB required
   - Fast, isolated test execution

2. AsyncMock for Async Operations
   - All manager methods are async
   - Using AsyncMock for proper async testing
   - pytest asyncio_mode = "auto"

3. Pydantic Model Validation
   - Tests verify Pydantic model construction
   - Invalid inputs would fail at model creation
   - Tests focus on business logic, not validation

4. Fixture Composition
   - Complex objects built from simpler fixtures
   - sample_control_details depends on sample_control
   - Promotes consistency across tests
