## Problem
The test `test_sanitize_replaces_we_recommend` in `tests/test_assessment.py` was failing because it had flawed assertion logic. The test expected `result == text` (unchanged input), but the `sanitize_text` function actually REPLACES forbidden phrases like "We recommend" with neutral language (e.g., "note:").

## Root Cause
- Line 2335 contained: `assert result == text`
- This assertion expected the sanitized text to be identical to the input
- However, the sanitize_text function is designed to replace "We recommend" with "note:" according to the _SANITIZATION_REPLACEMENTS dictionary in policy.py (line 165: `r"\bwe\s+recommend\b": "note:"`)

## Solution
Removed the incorrect assertion `assert result == text` from line 2335.

The test already had the correct assertion on line 2333:
```python
assert "we recommend" not in result.lower()
```

This properly validates that the sanitization removes the forbidden phrase without incorrectly asserting that the text remains unchanged.
